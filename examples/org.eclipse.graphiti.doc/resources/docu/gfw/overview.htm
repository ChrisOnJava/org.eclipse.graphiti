Graphics framework overview

 

The graphics framework supports the fast and easy creation of unified graphical tools based on GEF and connecting them with EMF models in Eclipse.

  • Have a look at the Graphiti introduction to get a first overview about the provided functionality and architecture.

·         

    • The plugin structure is a more technical chapter.
  • The scenario description explains the graphical editor we want to create in this tutorial.
  • Implementation of a graphical tool always starts with the creation of a diagram.
  • The features provided for a diagram define the functionality of the editor.

·         

    • The add feature defines which graphical elements are created for the domain elements.
    • The create feature supports the graphical creation of new domain elements (and their corresponding graphical elements).
    • The update feature defines how changes of the domain model are reflected in the graphical model.
    • The remove and delete feature support the deletion of graphical elements (and their corresponding domain elements).
    • The move feature provides the move functionality for the graphical shapes in the diagram.
    • The resize feature provides the resize functionality for the graphical shapes in the diagram.
    • The layout feature specifies how changes in the layout are reflected in the shapes.
    • The custom feature offers a hook to include generic functionality.
    • The add connection feature is used to add graphical connection representations for existing domain elements.
    • The create connection feature supports the graphical creation of new EReferences (and their corresponding graphical elements).

o    

      • Different types of connection anchors are supported.
      • It is possible to enhance the connection visualization further, e.g. with specific routings and decorators.
    • The direct editing feature allows changing values directly in the diagram.

o    

      • It is possible to immediately activate direct editing after object creation
    • The copy & paste feature supports copying model elements via the clipboard.
    • The drill-down functionality allows navigating from one diagram to associated diagrams.
  • The tool behavior provider is needed to integrate into the standard workbench tools.

·         

    • Context buttons allow to add actions as small, automatically appearing icons in the diagram.
    • Context menus allow to add actions in a menu appearing near the mouse pointer.
    • The palette provides tools to create connections and objects in the diagram.
    • Double-click behavior can be provided.
    • Rendering decorators can be used to display error-markers and other transient indicators.
    • The selection behavior can be adjusted, which is especially useful when using extended rendering areas.
  • It is easy to contribute to the property sheet.
  • You can provide own images to be used in the diagrams.
  • Styles can be used to provide a common look for shapes of the same type.
  • Color schemas provide complex, interaction-dependent background colors.
  • Polygon and polyline are the most flexible graphics algorithms for rendering shapes.
  • User defined properties can be attached, e.g. for simplified identification of pictogram elements.
  • A diagram can also be modified through features without having an editor opened using dark feature processing.
